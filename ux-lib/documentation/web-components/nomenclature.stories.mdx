# Nomenclature

Nous n'utilisons plus le prefixe data.
Nos composant députe par le prefixe ux. Exemple ux-svg.

## Taille

xxxs/xxs/xs/sm/md/lg/xl/xxl/xxxl

Le md est la taille normale

## Accessiblite

Pour l'accessiblité nous prefixons avec a11y.

# DEPRECIE Nomenclature

## Type de props/attributs

On utilise différents type de data pour nos WebComponents, chacun de ces type possède une nomenclature propre afin de les différencier:

## Data

> Proposition: `data-[attribut]`

- data-label
- data-sublabel
- data-label-left
- data-label-right
- data-date
- data-value (value que va prendre un input, un amount)
- data-value-min
- data-value-max
- data-step
- data-name (pour les inputs)
- data-currency (définir euros/dollars etc sur les composants input / amount)
- data-placeholder
- data-error
- data-error-msg
- data-icon (pour le nom de l’icône, ça semble suffisamment explicite)
- data-path
- data-link
- data-filename
- data-class (on peut avoir besoin de passer le nom d’une class sur le composant pour qu’elle soit reportée dans le DOM du component cf. `<bux-svg>`)
- data-content

Lorsqu’on a plusieurs data du même type à définir:

- data-primary, data-secondary ?
- data-left, data-right ?
- data-label-header?
- data-label-top-left
- data-label-top-right

**Attention a ne pas utiliser : **

- data-title

## Booléens

> Exemple d'utilisation :

- required
- disabled
- checked
- selected
- active
- editable
- visible

On gère aussi certain styles particuliers (appelé 'modifier' selon la nomenclature BEM)

- noborder

## Options de layout

> Proposition: layout=”grid”, layout=”columns-4”

Pour certains de nos composants on a besoin de passer une propriété de layout afin d’agencer leur contenu sans multiplier les composants. C’est le cas notamment pour les composants bux-card et bux-checkbox-group.
Problématique qui peut se poser est de savoir où on met le CSS correspondant à ces différents types de layout. Au niveau du composant pour que ceux-ci soient le plus indépendants possible? ou de façon génériques, on créé des class ‘objects’ selon la définition d’ITCSS, et on les réutilise dans les différents composants. (permet de limiter une redondance possible niveau CSS).
